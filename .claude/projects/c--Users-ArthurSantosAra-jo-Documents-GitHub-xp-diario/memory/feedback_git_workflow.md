description: Commits e push de branches são proibidos sem autorização explícita do usuário
metadata:
  type: feedback
---

Nunca fazer commit nem push de branches sem pedir autorização explícita ao usuário.

**Why:** O usuário definiu explicitamente essa regra — ele quer controlar o que vai para o repositório.

**How to apply:** Ao terminar um conjunto de mudanças, apresentar o feedback do que foi alterado, mas NÃO commitar. Pedir permissão antes de qualquer `git commit`, `git push` ou `git merge`.
