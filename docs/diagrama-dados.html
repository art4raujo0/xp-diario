Integrações Externas

Neon PostgreSQL 16

  • Provider: Neon serverless
  • Região: AWS sa-east-1 (São Paulo)
  • SSL: obrigatório (rejectUnauthorized: false)
  • Pool: pg.Pool com reconnect automático
  • Fallback: env LOCAL_DB_URL → DATABASE_URL

Render (Deploy)

  • Plataforma: Render Web Service
  • URL: https://xp-diario.onrender.com
  • Build: node backend/server.js
  • Variáveis: DATABASE_URL, JWT_SECRET, SMTP_*
  • Deploy automático via push na main

Nodemailer (Email)

  • Uso: recuperação de senha + notificações
  • Transport: SMTP com family: 4 (IPv4 forçado)
  • Porta: 587 (TLS)
  • Variáveis: SMTP_HOST, SMTP_USER, SMTP_PASS

JWT (Auth)

  • Expiração: 24 horas
  • Payload: us_id, us_nome, us_email, us_role
  • Middleware: verificarToken em rotas protegidas
  • Bloqueio: 5 tentativas falhas → us_bloqueado=true

Bcrypt (Senhas)

  • Rounds: 10 (default)
  • Uso: hash no cadastro, compare no login
  • Reset: token SHA-256, expira 1h, uso único

Express v5 (Backend)

  • Versão: 5.x (async error handling nativo)
  • Porta: 3000 (local) / Render (produção)
  • Static: /frontend servido direto
  • CORS: habilitado para desenvolvimento

Regras de Negócio Principais

RegraDetalhe
XP proporcional ao tempo1 minuto de estudo = 1 ponto XP. Calculado ao encerrar sessão ou registrar atividade manual.
Sessão única ativaApenas 1 sessão com status iniciada/pausada por usuário. Nova sessão retorna a existente.
Atividade mínimaSessões com menos de 1 minuto não geram registro em atividade nem somam XP.
Conquistas automáticasAvaliadas pelo ConquistasService ao registrar atividade ou encerrar sessão. Critérios: tempo total, streak, metas concluídas.
Disciplinas isoladasCada disciplina pertence ao usuário que a criou (di_usuario_id). Sem compartilhamento entre usuários.
Turmas por roleAluno entra via código. Professor cria e gerencia. Admin acessa tudo. Role no JWT.
Brute-force protection5 tentativas falhas de login bloqueiam a conta (us_bloqueado=true). Desbloqueio manual por admin.
Auditoria de tarefasToda mudança de status gera registro em tarefa_historico com timestamp.
